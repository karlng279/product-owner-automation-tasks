# ST-001: Acceptance Criteria

**Story:** View landing page
**USL Reference:** [ST-001 in usl.md](../usl.md#st-001-view-landing-page)
**Last Updated:** 2026-01-23

---

## UI Elements

- **AC-001:** Landing page displays a clear headline explaining the platform purpose (e.g., "Learn Incoterms the Easy Way").
- **AC-002:** Landing page shows a brief value proposition section (3-4 bullet points) explaining key features.
- **AC-003:** Landing page displays a hero section with a call-to-action button to start learning.
- **AC-004:** Main navigation menu is visible with links to: Learn, Quiz, Reference, Compare.
- **AC-005:** Footer displays a disclaimer stating content is for educational purposes only.

---

## UI Behavior

- **AC-006:** Clicking the "Start Learning" CTA button navigates to the Incoterms overview page.
- **AC-007:** Navigation links highlight the current active section.
- **AC-008:** Page renders within 1 second without visible layout shift.

---

## Logic

- **AC-009:** No authentication is required to access the landing page.
- **AC-010:** Landing page content is statically generated for fast load times.

---

## Special Notes

- **AC-011:** Page is responsive and displays correctly on desktop (1200px+), tablet (768px-1199px), and mobile (<768px).
- **AC-012:** All text meets WCAG AA contrast requirements.

---

## Non-Functional Requirements

- **NFR-001:** Page fully loads within 1 second for 95% of users.
- **NFR-002:** Page is SEO-optimized with proper meta tags, title, and description.

---

## Dependencies

*No dependencies - this is the entry point.*
