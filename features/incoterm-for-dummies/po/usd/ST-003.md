# ST-003: Acceptance Criteria

**Story:** View Incoterm detail page
**USL Reference:** [ST-003 in usl.md](../usl.md#st-003-view-incoterm-detail-page)
**Last Updated:** 2026-01-23

---

## UI Elements

- **AC-001:** Detail page displays the Incoterm code prominently (e.g., "FOB").
- **AC-002:** Page shows the full Incoterm name (e.g., "Free On Board").
- **AC-003:** Page includes a "Plain English" explanation section (3-5 paragraphs) written for beginners.
- **AC-004:** Page displays the applicable transport mode(s) clearly.
- **AC-005:** Page shows "When to use this Incoterm" guidance section.
- **AC-006:** Page includes "Common mistakes to avoid" section with 2-3 bullet points.
- **AC-007:** Breadcrumb navigation shows: Home > Learn > [Incoterm Code].
- **AC-008:** "Previous" and "Next" Incoterm navigation links are shown at the bottom.

---

## UI Behavior

- **AC-009:** Scrolling down reveals sections with smooth scroll behavior.
- **AC-010:** Clicking breadcrumb links navigates to respective pages.
- **AC-011:** Previous/Next links navigate to adjacent Incoterms in alphabetical order.

---

## Logic

- **AC-012:** Detail content is loaded from static JSON data based on Incoterm code in URL.
- **AC-013:** If an invalid Incoterm code is in URL, redirect to overview page with error toast.
- **AC-014:** Page URL follows pattern: /learn/[incoterm-code] (e.g., /learn/fob).

---

## Special Notes

- **AC-015:** All explanations avoid jargon; technical terms are explained inline or linked to glossary.
- **AC-016:** Include a "Key Point" callout box summarizing the most important takeaway.
- **AC-017:** Page includes a link to the official ICC definition (external link, opens in new tab).

---

## Non-Functional Requirements

- **NFR-001:** Page is server-side rendered for SEO.
- **NFR-002:** Page loads within 1 second.
