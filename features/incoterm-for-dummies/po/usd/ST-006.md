# ST-006: Acceptance Criteria

**Story:** Start Incoterm quiz
**USL Reference:** [ST-006 in usl.md](../usl.md#st-006-start-incoterm-quiz)
**Last Updated:** 2026-01-23

---

## UI Elements

- **AC-001:** Quiz landing page displays a welcoming header: "Test Your Incoterms Knowledge".
- **AC-002:** Page shows quiz description explaining format (multiple choice, number of questions).
- **AC-003:** "Start Quiz" button is prominently displayed.
- **AC-004:** Page displays estimated completion time (e.g., "5-10 minutes").
- **AC-005:** Optional difficulty selector: Beginner, Intermediate, Advanced.

---

## UI Behavior

- **AC-006:** Clicking "Start Quiz" navigates to the first question.
- **AC-007:** Selecting a difficulty level updates the quiz questions accordingly.
- **AC-008:** If no difficulty is selected, default to "Beginner".

---

## Logic

- **AC-009:** Quiz questions are loaded from static JSON data.
- **AC-010:** Beginner quiz has 10 questions, Intermediate has 15, Advanced has 20.
- **AC-011:** Questions are randomized each time the quiz is started.

---

## Special Notes

- **AC-012:** Include a brief tip: "Don't worry if you don't know all the answers - this is a learning tool!"
- **AC-013:** Progress is not saved if user leaves the page (no authentication).
- **AC-014:** Mobile view stacks elements vertically with full-width buttons.

---

## Non-Functional Requirements

- **NFR-001:** Quiz start page loads within 500ms.
- **NFR-002:** Question data preloads in background after page render.
