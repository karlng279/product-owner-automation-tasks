# ST-009: Acceptance Criteria

**Story:** Search Incoterms
**USL Reference:** [ST-009 in usl.md](../usl.md#st-009-search-incoterms)
**Last Updated:** 2026-01-23

---

## UI Elements

- **AC-001:** Search input field is displayed in the header/navigation area.
- **AC-002:** Search input has placeholder text: "Search Incoterms (e.g., FOB, Free On Board)".
- **AC-003:** Search icon is displayed inside or adjacent to the input field.
- **AC-004:** Clear button (X) appears when search input has text.

---

## UI Behavior

- **AC-005:** Search results appear in a dropdown as user types (autocomplete/typeahead).
- **AC-006:** Results appear after typing at least 2 characters.
- **AC-007:** Results update as user continues typing (debounced at 150ms).
- **AC-008:** Clicking a result navigates to that Incoterm's detail page.
- **AC-009:** Pressing Enter with text input navigates to search results page.
- **AC-010:** Pressing Escape closes the autocomplete dropdown.
- **AC-011:** Keyboard arrow keys navigate through autocomplete results.

---

## Logic

- **AC-012:** Search matches against:
  - Incoterm code (exact and partial: "FO" matches "FOB")
  - Full name (partial: "Board" matches "Free On Board")
  - Keywords/tags (e.g., "shipping", "insurance", "sea")
- **AC-013:** Search is case-insensitive.
- **AC-014:** Maximum 5 results shown in autocomplete dropdown.
- **AC-015:** If no results found, show "No Incoterms found for '[query]'" message.

---

## Special Notes

- **AC-016:** Search works offline (searches static JSON data client-side).
- **AC-017:** Autocomplete dropdown is accessible (ARIA labels, screen reader announcements).
- **AC-018:** Mobile: search expands to full width when focused.

---

## Non-Functional Requirements

- **NFR-001:** Autocomplete results appear within 100ms of typing pause.
- **NFR-002:** Search is fully functional without server requests (client-side only).
