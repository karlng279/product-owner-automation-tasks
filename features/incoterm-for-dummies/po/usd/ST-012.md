# ST-012: Acceptance Criteria

**Story:** View side-by-side comparison
**USL Reference:** [ST-012 in usl.md](../usl.md#st-012-view-side-by-side-comparison)
**Last Updated:** 2026-01-23

---

## UI Elements

- **AC-001:** Comparison table displays selected Incoterms as columns (2-3 columns).
- **AC-002:** Table rows compare the following attributes:
  - Full Name
  - Transport Mode
  - Risk Transfer Point
  - Cost Transfer Point
  - Seller's Key Responsibilities (bulleted)
  - Buyer's Key Responsibilities (bulleted)
  - Insurance Requirement
  - Best Used When (scenario guidance)
- **AC-003:** Column headers show Incoterm code prominently with option to remove.
- **AC-004:** "Add Another" button appears if fewer than 3 Incoterms selected.
- **AC-005:** Row labels are displayed in a sticky first column.
- **AC-006:** Visual indicator highlights key differences between terms (different values highlighted).

---

## UI Behavior

- **AC-007:** Clicking "X" on a column header removes that Incoterm from comparison.
- **AC-008:** Clicking "Add Another" scrolls/navigates back to selection area.
- **AC-009:** If only 1 Incoterm remains after removal, prompt to add more or view detail page.
- **AC-010:** Hovering over highlighted differences shows tooltip explaining the significance.
- **AC-011:** Table scrolls horizontally on mobile with sticky row labels.

---

## Logic

- **AC-012:** Comparison data is loaded from static JSON for each selected Incoterm.
- **AC-013:** Differences are automatically detected by comparing attribute values.
- **AC-014:** URL updates to reflect current comparison (shareable link).

---

## Special Notes

- **AC-015:** Print-friendly styles: comparison table fits on A4/Letter when printed.
- **AC-016:** "Which should I choose?" summary section below table provides guidance.
- **AC-017:** Link to each Incoterm's full detail page from the column header.
- **AC-018:** On mobile (< 768px), switch to card-based comparison (stacked, not table).

---

## Non-Functional Requirements

- **NFR-001:** Comparison table renders within 500ms.
- **NFR-002:** Horizontal scroll on mobile is smooth (60fps).
